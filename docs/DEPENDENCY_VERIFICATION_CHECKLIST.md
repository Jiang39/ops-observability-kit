# Ops Observability Kit - 依赖验证清单

最后更新：2026-03-02

## 使用规则
- 每项依赖必须有“验证动作 + 通过标准 + 证据记录”。
- 未通过时禁止将状态改为 `READY`。

## 依赖逐项验证
### DEP-O-001 本地运行时基础环境（非 Docker）
- 验证动作：执行 `.\scripts\start_local_stack.ps1`，检查 `.runtime/pids.json` 生成。
- 通过标准：脚本可执行并输出组件状态；Docker 非必选。
- 证据：启动脚本输出摘要 + `.runtime/pids.json` 路径记录。

### DEP-O-002 本地网络与端口
- 验证动作：执行端口占用检查（3000/9090/3100/4317 等）。
- 通过标准：关键端口无冲突或已完成重映射。
- 证据：端口检查结果与 `.env`/脚本端口配置一致。

### DEP-O-003 Grafana 数据源导入
- 验证动作：启动后检查 Prometheus/Loki/Tempo 数据源状态。
- 通过标准：无需手工添加即可查询。
- 证据：数据源列表截图说明或文本导出。

### DEP-O-004 示例日志输入
- 验证动作：写入测试日志并在 Explore 查询。
- 通过标准：5 分钟内命中日志，字段含 `service` 和 `traceId`。
- 证据：查询语句与命中结果摘要。

### DEP-O-005 试点服务窗口
- 验证动作：确认服务负责人、接入窗口、回滚窗口。
- 通过标准：`PILOT_SCOPE.md` 三项完整。
- 证据：文档链接 + 确认记录。

### DEP-O-006 告警接收通道
- 验证动作：发送测试告警到目标通道。
- 通过标准：通知在约定时延内送达。
- 证据：发送时间、送达时间、消息样例。
